---

## Why this exists

The Claude Code ecosystem is expanding quickly, but useful resources are scattered across repositories, docs, and community posts.  
This project collects the most practical and relevant tools in one place so you can discover, compare, and apply them faster.

---

## Key features

- Curated links focused on **real Claude Code workflows**
- Organized categories for quick discovery:
  - Skills
  - Hooks
  - Slash commands
  - Agent orchestrators

- Practical references to projects, documentation, and implementation examples

## Usage

### Browse on GitHub

1. Open: `https://github.com/adminsairolotech-bit/awesome-claude-code`
2. Use in-page search (`Ctrl/Cmd + F`) to jump to categories or tools
3. Follow links to external repositories, packages, and docs

### Browse locally

- Open `README.md` in your editor
- Use full-text search to filter by:
  - category names (e.g., “hooks”, “plugins”)
  - provider/tool names
  - use cases (e.g., testing, orchestration, productivity)

### Suggested workflow

- Start with your target category (e.g., **slash commands**)
- Shortlist candidates based on your use case
- Validate maintenance, license, and documentation in linked repos
- Integrate and test in a safe/dev environment before production use

## Contributing

Contributions are welcome and appreciated.

### What to contribute

- New high-quality Claude Code resources
- Better categorization or naming consistency
- Broken/outdated link fixes
- Clearer descriptions for existing entries

### Contribution guidelines

When submitting a resource, please ensure it is:

- Relevant to Claude Code usage/workflows
- Publicly accessible
- Useful, actionable, and not obvious spam
- Properly licensed (or clearly marked if license is unusual/unclear)
- Added to the most appropriate category

### How to contribute

1. Fork this repository
2. Create a branch: `feat/add-my-resource`
3. Edit `README.md` (and alternatives if needed)
4. Commit your changes with a clear message
5. Open a Pull Request with:
   - what was added/changed
   - why it’s valuable
   - any context reviewers should know

---

## Quality standards for entries

To keep this list high-signal, entries may be rejected if they are:

- Low effort, duplicated, or abandoned
- Off-topic for Claude Code
- Missing essential context/docs
- Legally or ethically problematic
